# Security Policy

## Reporting a vulnerability

Report vulnerabilities privately to the repository owner, Kallxy1/KallAntiKecot. Include affected route, impact, safe reproduction steps, and suggested mitigation. Do not open a public issue for an unpatched vulnerability.

Do not access, modify, retain, or disclose another user's data; perform denial-of-service testing; upload malware; or publish a vulnerability before remediation.

## Secrets

Never commit `.env`, Supabase service-role keys, GitHub tokens, Android keystores, OneSignal REST API keys, or Vercel credentials. Rotate any credential shared in chat, logs, screenshots, or commits.
